核心概念 · Headless 模式
📌 流程
PR 提交 → Claude 初审 → 人工终审 → Merge
Claude 初审：安全 / 质量 / 覆盖率
人工终审：业务逻辑 / 架构设计
💡 核心定义

通过 -p 参数，告诉 Claude 这是一次 非交互 调用

任务完成 → 直接输出结果并退出，无需人工输入

$ claude -p "Review this PR for security issues"
✓ 非交互 ✓ 自动执行 ✓ 直接输出
交互模式 vs Headless 模式
维度 🔵 交互模式 🟡 Headless 模式
界面 🖥️ 终端 TUI 界面 VS 无界面，仅 stdout
输入方式 💬 多轮对话交互 VS 一次性 Prompt / 管道
权限处理 🔐 人工确认弹窗 VS 自动跳过 / 白名单执行
会话管理 💾 自动保存会话 VS 默认不保存
人类角色 👤 审批者+检查者+控制者 VS ❌ 完全缺席
规则载体 🧠 实时人工判断 VS 参数硬编码预设
适用场景 🛠️ 开发 / 调试 / 探索 VS CI/CD / 自动化 / 批处理
⚠️ 人类三重角色（权限审批者 / 质量检查者 / 流程控制者）→ Headless 下全部缺席 → 必须预先硬编码

成本护栏 · 双保险
🔵 max-turns
$ claude -p "review" --max-turns 6

限制交互轮数（例：5 文件 + 结论 = 6 轮）

轮数耗尽 → 输出已有结论

0 turns6 / 10 turns
🟡 max-budget-usd
$ claude -p "review" --max-budget-usd 1.0

硬性金额上限（防 Token 累积超支）

预算触发 → 立即强制停止

$0limit $1.0
⚠️ 强制要求：生产环境务必同时配置，两道防线缺一不可
安全边界 · allowedTools
✅ 白名单 allowedTools · 推荐

仅允许明确列出的工具，其余一律拒绝

✓Read — 只读文件
✓Grep — 搜索内容
✓Bash(git log *) — 模式匹配
✓Glob — 文件查找
🚫 黑名单 disabledTools · 慎用

除禁止工具外，其余均可使用

×Edit — 禁止编辑
×Write — 禁止写入
×Bash(rm *) — 禁止删除
→ 未列出的工具均可使用 · 风险较高
⚠️ 严令禁止：dangerously-skip-permissions — 切勿忽视名称中的 dangerously，严禁在生产 CI 中使用

📄 System Prompt

✅--append-system-prompt 推荐

追加约束，保留基础能力

❌--system-prompt 覆盖

彻底替换默认行为，极易导致工具调用失败

追加 = 增强｜覆盖 = 风险
